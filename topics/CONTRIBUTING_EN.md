# Contributing Guide

Thank you for your willingness to contribute to Octoday Hub! We are committed to building a high-quality resource index for the embodied intelligence community, and every contribution helps more developers.

## How to Contribute

You can participate in the following ways:

- **Submit a Pull Request**: Modify or add content, following the PR template.
- **Submit an Issue**: Report errors, suggest adding new resources, or propose improvements.

## Contribution Workflow

### 1. Set Up Environment

- Make sure you have a GitHub account.
- Fork this repository to your account.
- Clone your fork locally (for bulk edits, you can also edit directly on the web).

### 2. Create a Branch

Create a new branch from `main`. Naming conventions:

- `add-company-[company-name]`
- `update-paper-[paper-title]`
- `fix-typo-[file-name]`

### 3. Make Changes

Please follow the formatting guidelines below:

#### Company List (01-companies.md)

- Provide **company name**, **official website link**, **one-sentence introduction**, **technical focus**, and **location**.
- Remain objective and neutral; summarize core business in one sentence.

#### Job Openings (02-jobs.md)

- Provide **job title**, **company**, **location**, **type** (full-time / internship / campus), and **application link**.
- Job postings must include a **deadline** or **last updated** date; please remove expired listings promptly.
- Use a unified table format and keep alignment consistent.

#### Paper Collection (03-papers.md)

- Provide **paper title**, **authors**, **venue / journal**, **year**, and **code link** (if available).
- Include a **one-sentence summary** of the core contribution when possible.
- Group by research direction (e.g., Manipulation, Locomotion, Navigation, etc.).

#### Tool Matrix (04-tools.md)

- Provide **tool name**, **type** (simulation / control / dataset / framework), **official link**, and a **one-sentence description**.
- Indicate whether it is open-source and the main programming language.

#### Cognitive Foundation (00-basics.md)

- For recommended books: provide **title**, **author**, **publication year**, and **reason for recommendation** (optional).
- For recommended courses: provide **course name**, **institution**, **link**, and **description**.

### 4. Submit a PR

- Ensure your changes are accurate and verifiable.
- One PR should focus on a single, clear change.
- Fill out the checklist in the PR template.

## Information Accuracy

- **Verifiable**: Information should be confirmable via official websites or authoritative sources.
- **Timely**: Please provide the latest information for time-sensitive content such as jobs and funding news.
- **Neutral**: Remain objective and avoid bias.

## Code of Conduct

- Be friendly and respectful.
- Respect intellectual property rights.
- For business collaborations, please contact us via email instead of posting in Issues or PRs.
